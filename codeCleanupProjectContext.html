📚 Project Documentation & Quick Start

1️⃣ Running the Interactive Web UI (Frontend)

We built a powerful Streamlit application to run the pipeline without writing any code.

# Activate the virtual environment
source venv/bin/activate

# Launch the Frontend Web UI
streamlit run app.py

This will open a browser window where you can upload datasets, tweak pipeline settings, and download the cleaned CSV instantly.

2️⃣ Running the Tagged Testing Suite

To guarantee a near-zero margin of error, the project features a massive 7-layer testing hierarchy. We use Pytest markers to track coverage.

# Run ALL Tests
pytest tests/

# Run ONLY Unit Tests
pytest tests/unit/

# Run a Specific Tag (e.g., Integration Pipeline Tests)
pytest -m IT_preprocessing_pipeline

3️⃣ Testing Folder Hierarchy & Tags

  • tests/unit/ - Tag: UT_... | Validates individual cleaning functions.
  • tests/integration/ - Tag: IT_... | Validates module chaining.
  • tests/system/ - Tag: ST_... | Validates end-to-end file parsing.
  • tests/regression/ - Tag: RT_... | Protects old logic from new bugs.
  • tests/performance/ - Tag: PT_... | Stress tests memory/speed constraints.
  • tests/validation/ - Tag: VT_... | Validates business schema rules.
  • tests/uat/ - Tag: UAT_... | Validates client-specific case studies.
